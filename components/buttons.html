Link

By default button and input elements are styled to look like .btn-primary. When possible, use button or input elements for buttons. If you use a link, please note that you can't assign role="button" to the element for it to act like a button. From Mozilla's documentation:

Warning: Be careful when marking up links with the button role. Buttons are expected to be triggered using the Space key, while links are expected to be triggered using the Enter key. In other words, when links are used to behave like buttons, adding role="button" alone is not sufficient. It will also be necessary to add a key event handler that listens for the Space key in order to be consistent with native buttons.

Types

Primary

This style is the most visually heavy of the other button styles and should be used to denote the primary action on a page and within a set of buttons (if applicable).

{% highlight html linenos %} {% endhighlight %}

Secondary

This indicates a positive, alternate action that could be engaged by the user within a workflow, page, or within a set of buttons.

{% highlight html linenos %} {% endhighlight %}

Confirmation

This button should be used to confirm a major action, typically within dedicated workflows such as completing a purchase (checkout), publishing an item, or updating an existing item.

{% highlight html linenos %} {% endhighlight %}

Warning

This should be used when the user needs warned a potential negative action such as removing a post, removing a team member, or deleting their account will be engaged if they click this button.

{% highlight html linenos %} {% endhighlight %}

Loaders / Icons

This is used to add icons into another button type or style.

{% highlight html linenos %} {% endhighlight %}

Styles

By default all button options have filled backgrounds with borders and an inner-shadow. Sometimes buttons need to be downplayed within a design to either help focus attention to a specific action, no action should be primary, or to not draw attention away from other content areas. In these instances, we have two button styles that can be applied to the options above.

Outlined

Like a secondary button in purpose, this stylistic variation is to be used when you desire an even simpler button style—especially when used in concert with the primary button style. This style works well within small UI actions areas that are tertiary in importance or when a larger button size would draw too much attention to the button.

{% highlight html linenos %} {% endhighlight %}

Clear

Sometimes called “link” buttons, this button style mimics the link style but with the ample click area buttons provide. This style is typically used within a set of actions and you want to downplay the importance of the action. Such as with a “Cancel” or “Discard” action. We don’t want to hide these actions, but we also don’t want to draw undue importance to them.

{% highlight html linenos %} {% endhighlight %}

Reversed

Most buttons appear on a light background within our product, yet there are instances where buttons may appear on a dark background. In these instances, you can invert the button colors.

{% highlight html linenos %} {% endhighlight %}

Ordering

To help maintain consistency throughout the product, buttons should maintain the following order arrangement:

Default

Most button groups should be ordered from the most important action to least important action, left to right.

{% highlight html linenos %} {% endhighlight %}

When a button’s word length and or container size makes it necessary to stack buttons on top of each other (like within a sidebar area), these buttons should be stacked from the most important action to the least important, top to bottom.

{% highlight html linenos %} {% endhighlight %}

Row

If a series of actions need to be placed in the same area as a title, the buttons here can be pulled to the right. Within this instance, the button order should be reversed, right to left.
