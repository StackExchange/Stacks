Banners

Inline banners are inserted with the user's area of focuse to convey pertinent information to the actions they have taken, need to take, or are about to take.

Informative

This style is used to communicate actionable information to the user. These are typically shown to inform about actions that need to be taken.

{% highlight html linenos %}

You need to be logged in to view this page.

Log in now

{% endhighlight %}

Success

This style is to be used only when we need to assure the user the action(s) they took were successful.

{% highlight html linenos %}

Your account was successfully created.

{% endhighlight %}

Your account was successfully created.

Error

This style is to inform the user that an error happened. Errors could be user-generated (incorrect data, missing fields, etc.) or system-generated. If the errors are user-generated, provide a clear set of instructions that helps the user properly address the issue. Vague error messages help no one.

{% highlight html linenos %}

You did not provide a valid email address. Please re-enter your email address.

{% endhighlight %}

You did not provide a valid email address. Please re-enter your email address.

Toast Notifications

Toasts are used either to quickly alert a user about an action's status that they just took or to inform the user about new items that are available. They are ephemeral, lasting only long enough for the user to read them. Copy should be kept extremely short (1-2 sentences at the most) and easily understood.
