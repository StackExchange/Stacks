--- layout: page title: Empty states description: Empty states are used when there is no data to show. Ideally they orient the user by providing feedback based on the the user’s last interaction or communicate the benefits of a feature. When appropriate, they should explain the next steps the user should take and provide guidance with a clear call-to-action. ---
{% header "h2", "No data" %}

Used when a user hasn’t interacted with a feature yet.

{% header "h3", "No data actionable" %}

If the user can take an action to fix the situation, it's best to draw attention to it, explain next steps, and provide a call-to-action.

{% highlight html %}
@Svg.Spot.EmptyXL.With("mb24")

Why is this blank? Explain it.

Call to action
{% endhighlight %}
{% spot, "EmptyXL", "mb24" %}

We haven’t received any data yet. Have you connected your Stack Overflow account?

Connect my account
{% header "h3", "No data non-actionable" %}

If the user can’t take an action to fix the situation, it’s more appropriate to set expectations.

{% highlight html %}
@Svg.Spot.EmptyXL.With("mb24")

{% endhighlight %}
{% spot, "EmptyXL", "mb24" %}

There’s no data associated with your account yet. Please check back tomorrow.
