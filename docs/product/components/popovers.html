--- layout: page title: Popovers description: Popovers are small content containers that provide a contextual overlay. They can be used as in-context feature explanations, dropdowns, or tooltips. ---
{% icon Wave %}

Help wanted! Each of our components will eventually ship with default JavaScript interactivity. We could use your help writing that JavaScript. To get started, check out the spec and a work in progress pull request.

For examples of some JavaScript that’s been shipped, see our sortable table component or our expandable control.

It can be tempting to hack together a quick fix in jQuery for displaying a Popover for your feature, but it may be just as much work as doing it properly as a Stimulus component—and this way every developer at Stack Overflow now has Popovers at their disposal.

{% header h2 | JavaScript data attributes %}
Attribute Applied to Description
data-controller="s-popover" Originating element Wires up the button to the popover controller
data-action="s-popover#toggle" Originating element Wires up the button to toggle the visibility of a generic popover.
aria-controls="[s-popover id]" Originating element Wires up the button to toggle the visibility of a specific popover with the ID provided.
id="[s-popover id]" .s-popover A unique id that the popover’s originating element can target.
{% header h2 | Examples %}

.s-popover--arrow__bc

Popover arrow appears bottom center

.s-popover--arrow__lb

Popover arrow appears left bottom

Popovers are invisible by default without any positioning applied to them. Instead, visibility and positioning of the popover is managed by JavaScript based on the arrow’s internal position. As shown in the example, an arrow with the .s-popover--arrow__bc positioning class will position the popover above the originating element. Visibility is also toggled by the originating element.
