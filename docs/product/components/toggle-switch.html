Class Description
.s-toggle Base class for the toggle component wrapper. All toggle switches must use this container class.
.s-toggle__control Class to apply to the actual toggle control element.
.s-toggle__switch Class for the visual switch element inside the control.
.s-toggle__knob Class for the movable knob element that slides from side to side.
.s-toggle--multiple Modifier class for toggle groups with multiple options.
.s-toggle__option Class for each option in a multiple toggle group.
.s-toggle__option--off Modifier class for "Off" or "None" options to use a neutral highlight state.
{% header "h2", "Examples" %} {% header "h3", "Single" %}

A single toggle is appropriate for simple "on/off" states eg. enabling emails, markdown editing, or various cookies. Toggle switches take up less space than an "on/off" radio button group and communicate their intended purpose more clearly than a checkbox that toggles functionality. They also provide consistency between desktop and mobile experiences.

{% header "h3", "3 or more options" %}

Toggles switches can be extended to choose between three or more states—in this case, the user is setting the frequency of an email digest. Use the multiple toggle instead of a radio group when unifying the aesthetics is important.
