{% header "h2", "Light, dark, and high contrast modes" %}

Stacks supports light, dark, and high contrast modes. By using Stacks, you will get each of these modes for free. By using a Stacks component, an atomic color class, or CSS variable directly, your theme will switch appropriately based on the following methods:

  1. You can apply .theme-system to the body element. This will change colors based on the prefers-color-scheme media query, which is ultimately powered by the user’s system or browser settings. This can be preferable for folks who have their system turn to dark mode based on ambient light or time of day.
  2. Alternatively, you can set a dark mode that is not system dependent by attaching .theme-dark to the body element.
  3. Adding .theme-highcontrast to the body element will boost colors to WCAG Level AAA contrast ratios in as many places as possible. This mode stacks on top of both light and dark modes. The only exception is branded themed colors remain untouched by high contrast mode.

There are also conditional classes that can be applied to override assumed dark mode colors, force light mode, or to force dark mode. Forcing modes can be good for previews in admin-only situations.

{% header "h2", "Theming API" %}

Stacks provides an API to apply custom themes.

{% header "h3", "Programmatic theme generation" %}

Use .create-custom-theme-hsl-rgb-variables(@color, @tier, @modeCustom) to create a custom theme.

This function generates two sets of CSS variables: 1) independent h/s/l and r/g/b color variables and 2) variables at each colors stop that reference the h/s/l and r/g/b variables. Provide this function the arguments defined below to generate theme colors which will apply across Stacks.

Argument Type Default Description
@color HSL, hex, or other color value Color to use to generate theme values.
@tier primary | secondary primary Color tier to generate.
@modeCustom base | dark base The color mode the theme applies to.
{% highlight css %} .theme-custom.themed { .create-custom-theme-hsl-rgb-variables(hsl(172, 37%, 48%), primary); .create-custom-theme-hsl-rgb-variables(hsl(259, 29%, 55%), secondary); .create-custom-theme-hsl-rgb-variables(hsl(201, 70%, 55%), primary, dark); .create-custom-theme-hsl-rgb-variables(hsl(270, 34%, 40%), secondary, dark); } {% endhighlight %}
{% header "h4", "Output" %}
{% highlight css %} /* Input */ .theme-custom.themed { .create-custom-theme-hsl-rgb-variables(hsl(172, 37%, 48%), primary); } /* Output */ .theme-custom.themed { /* HSL variables */ --theme-base-primary-color-new-h: 172; --theme-base-primary-color-new-s: 37%; --theme-base-primary-color-new-l: 48%; /* Color variables based on HSL and RGB variables */ --theme-primary-new-custom: var(--theme-primary-new-custom-400); --theme-primary-new-custom-100: hsl(var(--theme-base-primary-color-new-h), calc(var(--theme-base-primary-color-new-s) + 0 * 1%), clamp(70%, calc(var(--theme-base-primary-color-new-l) + 50 * 1%), 95%)); --theme-primary-new-custom-200: hsl(var(--theme-base-primary-color-new-h), calc(var(--theme-base-primary-color-new-s) + 0 * 1%), clamp(55%, calc(var(--theme-base-primary-color-new-l) + 35 * 1%), 90%)); --theme-primary-new-custom-300: hsl(var(--theme-base-primary-color-new-h), calc(var(--theme-base-primary-color-new-s) + 0 * 1%), clamp(35%, calc(var(--theme-base-primary-color-new-l) + 15 * 1%), 75%)); --theme-primary-new-custom-400: hsl(var(--theme-base-primary-color-new-h), calc(var(--theme-base-primary-color-new-s) + 0 * 1%), clamp(20%, calc(var(--theme-base-primary-color-new-l) + 0 * 1%), 60%)); --theme-primary-new-custom-500: hsl(var(--theme-base-primary-color-new-h), calc(var(--theme-base-primary-color-new-s) + 0 * 1%), clamp(15%, calc(var(--theme-base-primary-color-new-l) + -14 * 1%), 45%)); --theme-primary-new-custom-600: hsl(var(--theme-base-primary-color-new-h), calc(var(--theme-base-primary-color-new-s) + 0 * 1%), clamp(5%, calc(var(--theme-base-primary-color-new-l) + -26 * 1%), 30%)); } {% endhighlight %}
{% header "h4", "Manual addition of theme variables" %}

If you need to apply a theme without using the above function, you can do so by manually adding the variables above to your CSS. The most common use for this approach is when the theme needs to change client-side, such as when allowing the user to change and preview a theme dynamically.

With this approach, we recommend targeting new h/s/l and r/g/b color variables on a parent element that includes .themed class.
