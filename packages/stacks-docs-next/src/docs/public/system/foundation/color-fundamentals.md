## Palette

### Neutral colors

<p>The neutral palette consists of black, white, and grays. The neutral palette is dominant in our UI, using subtle shifts in value to create hierarchy and organize content.</p>

## Light, dark, and high contrast modes

<p>Stacks supports light, dark, and high contrast modes. By using Stacks, you will get each of these modes for free. By using a Stacks component, an atomic color class, or CSS variable directly, your theme will switch appropriately based on the following methods:</p>

<ol>
    <li>You can apply <code>.theme-system</code> to the <code>body</code> element. This will change colors based on the <code>prefers-color-scheme</code> media query, which is ultimately powered by the user's system or browser settings. This can be preferable for folks who have their system turn to dark mode based on ambient light or time of day.</li>
    <li>Alternatively, you can set a dark mode that is not system dependent by attaching <code>.theme-dark</code> to the <code>body</code> element.</li>
    <li>Adding <code>.theme-highcontrast</code> to the <code>body</code> element will boost colors to WCAG Level AAA contrast ratios in as many places as possible. This mode stacks on top of both light and dark modes. The only exception is branded themed colors remain untouched by high contrast mode.</li>
</ol>

<p>There are also <a class="s-link" href="/product/develop/conditional-classes#dark-mode">conditional classes</a> that can be applied to override assumed dark mode colors, force light mode, or to force dark mode. Forcing modes can be good for previews in admin-only situations.</p>
