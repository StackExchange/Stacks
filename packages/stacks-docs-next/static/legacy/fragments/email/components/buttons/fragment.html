Buttons are the primary way for users to take action from an email. Buttons should have ample click / tap space and describe their actions. Creating a button that displays consistently across email clients requires multiple HTML tags.

Styles

Section titled Styles

A button’s visual weight should correspond to the importance of the button’s action. The more important the action, the heavier the button’s visual weight.

Filled

Section titled Filled

A visual style used to highlight the most important actions. To avoid confusing users, don’t use more than one filled button within an email section.

Note: Using the s-btn class with s-btn__filled, s-btn__white or s-btn__outlined applies :hover progressive enhancements defined in the <style> tag. Only email clients that support <style> in <head> will render the :hover.
