{% header "h3", "Checkmark" %}

Use the checkmark variant for selection items within menus and dropdowns (vertical lists only). Its recommended use is for single-selection (like a radio), but it can be configured for multi-select. This variant is stylized for this specific context and should not be used for lists of options in a form (use radio buttons or checkboxes instead).

{% tip, "", "mb32" %} Note on focus: This variant does not have a dedicated focus state. Focus should be handled at the parent level, such as on the full menu item that contains the checkmark. {% endtip %}

{% header "h2", "Accessibility" %}

The best accessibility is semantic HTML. Most screen readers understand how to parse inputs if they’re correctly formatted. When it comes to checkboxes, there are a few things to keep in mind:

  • All inputs should have an id attribute.
  • Be sure to associate the checkbox/radio label by using the for attribute. The value here is the input’s id.
  • If you have a group of related checkboxes or radios, use the fieldset and legend to group them together.

For more information, please read Gov.UK’s article, "Using the fieldset and legend elements".
