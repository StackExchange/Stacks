Icons approach

Our recommended way for including icons is called inline SVG. Which basically means we just paste SVG code in view files. And yes, we prefer that way than using CSS background-image (unless there's not other option).

It's not super easy but the good news is we have it automated. There's helper you can use in order to include one of existing icons in code. Here's how it works:

{% capture html %}
@Svg.Search @Svg.Social.Facebook
{% endcapture %}{% include example.html html=html preview=false %}

Here's pro CSS tip. If you want your icon to have a color of parent element (so basically behave like text) set this styling:

{% capture css %} [role="icon"] { fill: currentColor; } {% endcapture %}{% include example.html html=css preview=false %}

Adding own icons

If there's icon that doesn't exist in library above, you probably will need to add it by yourself. Few things to remember:

  • It has to be SVG.
  • It has to be smaller than 4kb.
  • SVG code has to be optimized. Here's instruction how to do it.
  • SVG element should have at least those attributes: role="icon" (we also use it as selector in LESS), width="", height="", viewBox="". Rest can be skipped.

Ok, now you can add your icon. Here's how:

  1. Move your optimized SVG file to following folder: */Content/Img/svg-icons/.
  2. You can create subfolders to categorize icons. If you create file bar.svg in folder foo your helper will look like this @Svg.Foo.Bar.
  3. Edit */Helpers/Svg.cs, copy this template: public static IHtmlString SvgName { get; } = GetImage(nameof(SvgName)); and paste it (updated of course) in appropriate place in that file (or simply follow the guidelines in that file).

Optimize your SVG

When you export your SVG file it will probably contain a lot of crap code.

  • There's online tool to optimize SVGs called SVGOMG - you can upload file or paste code. In settings panel I usually have all checkboxes checked except:
    • Show original - well...
    • Prefer viewBox to width/height - we actually prefer both.
    • Remove viewBox.
    • Also, I usually set Precision to 3 or 4.
  • Illustrator is pretty good with optimizing SVGs but I still recommend to run them through SVGOMG.
  • Sketch does bad job with optimizing SVGs however there's really great plugin (based on the same library used by SVGOMG) to make it better. It's called SVGO Compressor - I highly recommend installing it (you may still want to run exported files through SVGOMG).

Even after optimizing SVG you still should open it in your code editor and make sure that:

  • It (SVG element) does have role="icon" attribute. We use it in multiple places as a selector in LESS.
  • It does have viewBox, width and height attributes.
  • It doesn't have any whitespaces (you can remove all line breaks etc.)
  • It doesn't have xmlns attribute in SVG element.
  • It doesn't have any comments.
  • It doesn't have any unnecessary groups (G elements).
  • It doesn't have any unnecessary styling, transformations or masks.
  • It doesn't have fill attributes.
